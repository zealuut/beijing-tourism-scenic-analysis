# Beijing Tourism Scenic Analysis

北京旅游景区评论、治理主题识别与案例解释项目。

当前仓库围绕北京 20 个景区，形成了一条从评论文本识别到治理诊断、再到重点主题动态量化和典型景区案例分析的完整链路。仓库当前包含：

- 评论样本整理与实验脚本
- LDA 一级治理主题结果
- LLM 11 个二级治理问题标签结果
- SnowNLP 连续/离散情感结果
- 论文口径 IPA 与聚类结果
- 情感得分与治理标签回归底表与脚本
- 交通主题时间序列建模数据与脚本
- 北京动物园 POI、动态回归与多元时间序列案例材料

## 目录结构

- `scripts/`
  项目实验脚本，统一使用 `NN_run_experimentNN_*.py` 或 `NN_build_experimentNN_*.py` 命名。
- `docs/`
  方法说明、指标口径、案例分析与全流程文档。
- `dicts/`
  标签与规则字典。
- `prompts/`
  LLM 标注 prompt 与 few-shot 示例。
- `data/`
  项目中间数据、建模底表与原始专题数据。
- `outputs/`
  IPA、聚类、案例分析与时间序列模型输出。

## 当前主线

建议按研究逻辑而不是文件编号顺序理解本项目：

1. 原始评论整理与统一样本构建
2. LDA 一级主题与 LLM 二级问题标签识别
3. SnowNLP 连续/离散情感评估
4. 论文口径 IPA 与聚类，识别重点治理主题和高紧迫景区
5. 情感得分与 11 个二级标签回归，检验治理问题与情绪结果的关系
6. POI、交通主题时间序列与北京动物园综合案例解释
