# 11 个二级标签边界说明

## 1. 文档目的

本文件用于当前 `旅游景点分析` 项目的评论级 LLM 打标。

- 适用语料：`experiment2_lda_评分分层300条汇总大表.csv`
- 适用任务：只判断一条评论属于哪些治理问题标签
- 不包含任务：不做情绪分类，不做一级主题判断

## 2. 统一判标原则

1. 只有当评论里出现了明确的抱怨、摩擦、障碍或治理问题时，才打二级标签。
2. 纯赞美、纯攻略、纯景观描述、纯知识介绍，即使提到门票、交通、导游等词，也可以不打标签。
3. 一条评论可以有多个标签；只要问题都明确出现，就允许多标签。
4. 不因为出现某个关键词就机械打标，要看评论的核心抱怨落点。
5. 优先区分相邻边界，而不是追求“所有提到的名词都打上”。

## 3. 高混淆边界

### 3.1 `staff_service` vs `service_process`

- `staff_service`：抱怨工作人员、客服、检票员、讲解员的态度、耐心、回应、敷衍、推诿。
- `service_process`：抱怨组织安排、集合衔接、安检检票流程、执行效率、动线安排本身。

### 3.2 `reservation_entry` vs `platform_transaction`

- `reservation_entry`：景区预约、实名、入园规则、检票核验、入口规则、二维码入场、预约说明不清。
- `platform_transaction`：第三方平台或 App 的订单、退票退款、出票、核销失败、二维码失效、平台客服履约问题。

### 3.3 `queue_wait` vs `crowding`

- `queue_wait`：重点是“等了多久”“队伍很长”“排了很久”。
- `crowding`：重点是“人太多”“太挤”“空间拥堵”“人流压力大”。

### 3.4 `commercialization` vs `staff_service`

- `commercialization`：重点是卖货、商拍、旅拍、强推消费、纪念品/餐饮/文创干扰游览。
- `staff_service`：重点是态度差、爱答不理、沟通粗暴。

## 4. 11 个标签口径

### `staff_service`
- 中文名：现场人员服务态度
- 判定要点：评论工作人员、客服、检票员等人员的态度、耐心、响应意愿与现场帮助程度。
- 不含：单纯流程慢、组织混乱；这类优先看 `service_process`。

### `service_process`
- 中文名：服务流程与组织衔接
- 判定要点：评论集合安排、动线组织、安检检票流程、现场衔接、执行效率是否顺畅。
- 不含：只是抱怨排队时间长；那更偏 `queue_wait`。

### `guide_explanation`
- 中文名：讲解与导览质量
- 判定要点：评论导游、讲解员、语音导览的专业度、清晰度、信息量、节奏与表达效果。
- 不含：只是说“有导游/有讲解”；没有明确抱怨时不打。

### `queue_wait`
- 中文名：排队等待
- 判定要点：评论排队、等候、等待时间过长，以及由排队造成的体验损失。

### `reservation_entry`
- 中文名：预约购票与入园核验
- 判定要点：评论预约、抢票、实名、二维码、入口、入场、检票、核验、入园规则本身的问题。

### `ticket_price`
- 中文名：门票价格与性价比
- 判定要点：评论门票价格、联票、附加收费、是否太贵、是否不值、性价比不合理。

### `traffic_access`
- 中文名：交通到达与停车换乘
- 判定要点：评论地铁、公交、打车、停车、换乘、找路、到达景区是否方便。

### `facility_hygiene`
- 中文名：配套设施与环境卫生
- 判定要点：评论厕所、卫生间、座椅、休息区、垃圾、异味、基础设施、环境清洁等问题。

### `crowding`
- 中文名：拥挤度与人流压力
- 判定要点：评论景区内部人太多、太挤、人山人海、空间拥堵、嘈杂压迫感。

### `commercialization`
- 中文名：商业化与消费干扰
- 判定要点：评论景区卖货、商拍、旅拍、强推项目、纪念品、文创、餐饮等对游览体验的干扰。

### `platform_transaction`
- 中文名：平台交易与履约
- 判定要点：评论携程、Trip.com、第三方平台、订单、退款、出票、核销、二维码失效、平台客服履约问题。

## 5. 空标签规则

以下情况允许 `issue_tags = []`：

- 纯正向评价，没有明确治理问题
- 纯中性攻略或景观介绍
- 只是提到门票、交通、导游等名词，但没有抱怨
- 只是建议“早点去”“值得去”，但没有明确问题

## 6. 当前项目落地方式

- 一级主题：由 LDA 负责抽取
- 二级标签：由 LLM 按本文件定义做评论级多标签标注
- 后续聚合：按景区与标签占比汇总，支撑一级主题解释和治理分析
