# 多元时间序列动态案例方法

## 1. 这一步放在全文里的位置

这一步不是替代前面的 20 景区横截面分析，而是作为：

- `北京动物园案例深化`
- `票务预约与入园体验的动态补充解释`

前面的 LDA、LLM 标签、IPA、聚类和 POI 已经回答了：

- 哪类问题最突出；
- 哪些景区更需要治理；
- 北京动物园为什么属于 `票务高压治理型`。

而这一步进一步回答：

**在北京动物园已经存在较强票务/入园治理压力的前提下，这种前端压力在什么条件下更容易升高。**

## 3. 为什么不直接沿用旧的 Meta+LSTM+Attention 主口径

已知信息是：

- 旧版 `Meta + LSTM + Attention` 没有通过统计显著性检验。

因此本文不再把它作为正式主模型，原因有三：

1. 它不适合承担“统计显著性结论”  
   attention 热力图可以很好看，但不能代替显著性检验。

2. 当前论文更需要解释，不是单纯追求预测黑箱  
   你们主线是治理诊断，不是算法竞赛。

3. 这份时间序列数据的使命是服务全文叙事  
   它应该补充“北京动物园何时更容易出现票务前端压力”，而不是另起一条复杂模型线。

## 4. 正式主模型怎么定

### 4.1 目标变量

正式目标变量为：

- `ticket_index`

本文将其解释为：

**票务与入园前端压力的动态代理量**

它不是实际排队人数，也不是实际客流，而是反映：

- 门票关注度
- 入园前端需求压力

### 4.2 正式保留变量

正式主模型只保留这些变量：

- `log_zoo_index`
- `aqi_good_z`
- `is_weekend`
- `is_holiday`
- `is_school_holiday`
- `log_ticket_lag1`
- `log_ticket_lag7`

其中：

- `log_zoo_index`：总体景区关注度，表示潜在需求强弱
- `aqi_good_z`：AQI 反向标准化，值越高表示天气条件越好
- `is_weekend`：周末效应
- `is_holiday`：法定节假日冲击
- `is_school_holiday`：学生假期冲击
- `log_ticket_lag1`：前一天票务压力惯性
- `log_ticket_lag7`：前一周同日票务压力惯性

### 4.3 不直接进入主模型的列

这些列不直接进入正式主模型：

- `day_of_week`
- `diff_1`
- `diff_7`
- `rolling_mean_7`
- `history_error`

原因是：

1. `day_of_week` 与 `is_weekend` 信息重叠
2. `diff_1 / diff_7 / rolling_mean_7 / history_error` 更偏工程特征
3. 这些变量写进论文时解释性不如 `lag1 / lag7` 直观
4. 它们容易与正式滞后项高度相关，影响口径稳定性

## 5. 正式方法

正式主方法采用：

- `OLS 动态回归`
- `Newey-West / HAC 稳健标准误`

这么做的原因是：

- 可以处理时间序列中的自相关问题
- 可以给出稳健标准误和显著性检验
- 结果比黑箱模型更适合论文正文

模型写法可以表述为：

`log(ticket_index_t) = β0 + β1 log(zoo_index_t) + β2 AQI_good_z_t + β3 is_weekend_t + β4 is_holiday_t + β5 is_school_holiday_t + β6 log(ticket_index_{t-1}) + β7 log(ticket_index_{t-7}) + ε_t`

## 6. attention 还能不能留

能留，但角色要变。

本文不再把 attention 当作正式主模型的显著性来源，而是把它降级为：

- `辅助可视化层`
- `叙事增强层`

具体做法是：

1. 构造过去 14 天的 `ticket_index` 与 `zoo_index` 滞后序列
2. 用线性动态模型估计这些时滞信息对当天 `ticket_index` 的贡献
3. 将各日的滞后贡献绝对值做 softmax 归一化
4. 得到一个 `attention 风格时滞热力图`

这个热力图回答的是：

- 高压日更依赖最近哪几天的信息
- 常态日的时滞依赖是否更平缓

它**不承担显著性检验**，只承担：

- 图像展示
- 机制辅助解释

## 7. 这一步如何服务全文叙事

这样处理后，全文逻辑会变得很顺：

1. 横截面部分  
   北京动物园被识别为 `票务高压治理型`

2. POI 部分  
   北京动物园具有高到达汇聚和高周边活动环境

3. 动态时间序列部分  
   当总体景区关注度上升，且近期票务压力已经抬升时，北京动物园的票务前端压力更容易继续升高

也就是说，这一步不再是“另做一个炫模型”，而是用来补一句：

**北京动物园不仅为什么会有票务前端压力，而且这种压力何时更容易被放大。**

## 9. 论文里建议怎么写

推荐口径是：

> 为进一步补充北京动物园票务与入园前端压力的动态机制，本文在横截面治理诊断之外，引入北京动物园日度多元时间序列数据，构建可解释动态回归模型。本文将“北京动物园门票”百度指数视为票务前端压力的动态代理量，并将景区总体关注度、天气条件、周末与假期因素以及短期滞后项纳入统一框架进行分析。考虑到旧版 Meta+LSTM+Attention 模型未能提供稳健的统计显著性结果，本文不再将其作为正式主模型，而仅保留 attention 风格的时滞热力图作为辅助可视化，用于展示高压日对近期时滞信息的依赖结构。

- 若需要和 POI 空间解释、多元 Meta-LSTM-Attention 探索模型一起组织为最终案例部分，建议同步参考 `北京动物园综合案例分析说明.md`。
