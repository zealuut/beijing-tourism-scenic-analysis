# LDA 一级主题与 11 个二级标签对应说明

## 1. 这份文档解决什么问题

本项目同时有两套文本结果：

- `LDA 一级主题`
- `LLM 11 个二级问题标签`

两者不是同一层级，也不是一一对应关系。

这份文档的目的，是把它们之间的关系说清楚，避免后续在论文里把一级主题和二级标签混写。

## 2. 两套结果分别做什么

### 2.1 LDA 一级主题

LDA 的作用是把评论聚合成较粗的“问题领域”或“体验主题”，用于：

- 做整体主题识别
- 做景区层 IPA
- 做聚类分析

当前正式使用的一级主题有 4 个：

1. `票务预约与入园体验`
2. `现场服务与导览体验`
3. `交通与开放空间体验`
4. `整体满意与感知评价`

其中第 4 个是满意主题，不进入治理 IPA 与治理聚类。

### 2.2 LLM 11 个二级问题标签

11 个标签的作用，是把“具体抱怨点”标出来，用于：

- 解释一级主题内部到底在抱怨什么
- 做评论级多标签识别
- 做景区问题结构补充说明

因此，项目里的层级关系是：

`LDA 一级主题 -> LLM 二级问题标签`

## 3. 一级主题和二级标签怎么对应

不是一一对应，而是**多对多**关系。

原因有 3 个：

1. LDA 一级主题是按词共现结构形成的主题簇，强调“评论主要在谈什么”。
2. LLM 二级标签是按治理问题边界打的标签，强调“评论具体在抱怨什么”。
3. 一条评论在 LDA 中只会落到一个主主题，但在 LLM 标签里可以同时属于多个标签。

所以正确理解应当是：

- 一级主题决定这条评论属于哪个较粗的问题领域
- 二级标签解释这个问题领域里具体出现了哪些治理摩擦

## 4. 当前论文口径下的概念对应

### 4.1 `票务预约与入园体验`

概念上最核心的二级标签是：

- `queue_wait`
- `reservation_entry`
- `ticket_price`
- `platform_transaction`

在当前样本里，这个主题也经常外溢到：

- `traffic_access`
- `crowding`

解释：

- 很多游客在谈预约、购票、核验、排队时，会顺带提到拥挤、到达、停车、换乘等前后链路问题。
- 因此这个主题不是狭义“买票”，而是更广义的“票务-预约-入园链条体验”。

### 4.2 `现场服务与导览体验`

概念上最核心的二级标签是：

- `guide_explanation`
- `staff_service`
- `service_process`

次级可能外溢到：

- `reservation_entry`
- `ticket_price`
- `platform_transaction`

解释：

- 当游客在抱怨讲解、导游、导览设备、工作人员态度、现场衔接时，常常会顺带提到收费、语音导览购买、流程指引等问题。
- 因此这个主题对应的是“现场服务与导览”这一组体验，不是单独只等于某一个标签。

### 4.3 `交通与开放空间体验`

概念上最核心的二级标签应当是：

- `traffic_access`
- `crowding`
- `facility_hygiene`

解释：

- 这个主题更接近“到达、空间秩序、人流密度、基础环境”的组合体验。
- 其中 `facility_hygiene` 在这里不是狭义厕所卫生，而是与开放空间体验相关的环境与设施支撑。

需要注意：

- 当前 6000 条样本里，这个主题的 `issue_review_n` 只有 `26`，二级标签共现样本偏少。
- 因此这个主题和 11 标签的经验对应暂时不如前两个主题稳定。
- 论文里解释这个主题时，应优先结合 LDA 关键词、代表评论和景区语境，而不要只依赖标签频次排序。

### 4.4 `整体满意与感知评价`

这个主题是满意主题，不进入治理 IPA。

它与 11 个二级标签没有固定绑定关系。  
当它内部出现问题标签时，表示：

- 这条评论整体在做满意/总体评价
- 但评论中仍顺带提到了具体问题

所以它更适合作为：

- 情绪与总体感知层

而不是：

- 一级治理主题层

## 5. 当前样本的实证对应结果

基于当前样本，我另外导出了一张实证对照表：

- `旅游景点分析/outputs/ipa_paper/experiment7_theme_issue_tag_mapping.csv`

这张表的统计口径是：

- 先用 `experiment3_lda_topic_assignments.csv` 取得每条评论的一级主题
- 再用 `experiment4_llm_issue_output_merged.csv` 取得每条评论的 11 个二级标签
- 按一级主题统计每个标签在该主题中的出现次数与占比

表中主要字段含义：

- `review_n`：该一级主题下的评论数
- `issue_review_n`：该一级主题下被判定为有问题的评论数
- `tag_review_n`：该一级主题下命中某个二级标签的评论数
- `tag_share_in_theme_reviews`：某标签在该主题全部评论中的占比
- `tag_share_within_issue_reviews`：某标签在该主题“有问题评论”中的占比

## 6. 当前样本里最值得看的实证对应

### 6.1 `现场服务与导览体验`

当前样本中，这个主题最稳定，对应关系也最清楚。  
在其有问题评论中，占比最高的标签是：

- `guide_explanation`：`0.5263`
- `ticket_price`：`0.1895`
- `crowding`：`0.1789`
- `staff_service`：`0.1579`
- `service_process`：`0.1053`

解释上应把它理解为：

- 以 `guide_explanation / staff_service / service_process` 为主
- 其余标签是混合评论带来的外溢

### 6.2 `票务预约与入园体验`

当前样本中，这是最强的治理主题。  
在其有问题评论中，占比最高的标签是：

- `ticket_price`：`0.2617`
- `crowding`：`0.1963`
- `traffic_access`：`0.1776`
- `queue_wait`：`0.1472`
- `reservation_entry`：`0.1449`
- `platform_transaction`：`0.1168`

解释上应把它理解为：

- 核心仍是 `queue_wait / reservation_entry / ticket_price / platform_transaction`
- `crowding` 与 `traffic_access` 是当前样本里很常见的链路外溢问题

### 6.3 `交通与开放空间体验`

这个主题当前样本里的问题评论数较少，所以频次排序不够稳定。  
在现有样本中，占比靠前的是：

- `ticket_price`：`0.4231`
- `crowding`：`0.3077`
- `reservation_entry`：`0.1154`

这里不能机械理解为“交通主题其实等于票价主题”。  
更合理的解释是：

- 该主题当前边界还偏松
- 少量混合评论会把票务抱怨一并带入
- 因此这个主题应以概念边界和代表评论解释为主，以标签频次为辅

## 7. 论文里应该怎么写

推荐写法是：

1. 先说明 LDA 用于抽取一级主题，识别较粗的问题领域。
2. 再说明 LLM 11 标签用于二级治理解释，识别评论中的具体摩擦点。
3. 明确指出两者是“一级主题-二级标签”的层级关系，而不是一一对应关系。
4. 在解释每个一级主题时，同时报告其概念主标签和当前样本中的高频共现标签。

## 8. 当前使用建议

- 做 IPA 和聚类时，以 `LDA 一级主题` 为主。
- 做主题内部解释时，以 `11 个二级标签` 为辅。
- 不建议直接把某个一级主题机械映射成单个二级标签。
- `交通与开放空间体验` 当前边界相对更弱，解释时要更谨慎。
