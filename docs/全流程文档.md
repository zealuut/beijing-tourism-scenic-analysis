# 项目全流程文档

## 1. 项目目标

本项目不是单纯做“景区评论情感分析”，而是基于北京 20 个景区的游客评论，构建一套兼顾主题识别、情绪评估、治理诊断与典型案例解释的景区治理分析框架。

当前正式主线包括四类工作：

1. 评论文本识别  
   从游客评论中提取一级治理主题与二级治理问题标签。
2. 情感评估与诊断  
   生成连续/离散情感结果，并通过 IPA、聚类和回归识别治理重点。
3. 空间与动态解释  
   用 POI、交通拥堵指数和日度多元时间序列补足治理压力的空间与时间机制。
4. 重点景区案例  
   以北京动物园为综合案例，对高治理压力景区进行更细的解释和比较。

## 5. 阶段一：评论样本与主题识别

对应底表：

- `data/regression/sentiment_regression.xlsx`

研究目的：

- 使用连续型情感得分与离散型情感类别，检验 11 个二级治理标签与情绪结果之间的关系；

- 连续情感得分部分采用多元线性回归；
- 离散情感类别部分采用逻辑回归口径，当前脚本使用多项逻辑回归；
- 自变量均为 11 个二级标签哑变量。

主要输出目录：

- 用 attention 机制缓解普通 LSTM 对远期信息影响衰减过快的问题；
- 作为交通主题的深度学习补充模型。

## 10. 阶段六：北京动物园综合案例

### 10.1 案例定位

北京动物园是当前研究中治理紧迫度较高、且在票务预约与入园体验主题上压力更突出的典型景区，因此被用作综合案例。

### 10.2 正式解释链路

综合案例的正式解释链路由三部分组成：

1. `实验11`  
   POI 空间环境解释，回答“为什么治理压力容易前置到入口前端”。
2. `实验12`  
   可解释动态回归，回答“在什么条件下票务前端压力更容易继续升高”。
3. `实验17`  
   多元 Meta-LSTM-Attention 探索模型，补充更强预测口径下的动态模式比较。

### 10.3 案例专题数据

对应原始底表：

- `data/time_series/beijing_zoo/beijing_zoo_multivariate_meta_lstm_input.xlsx`
- `data/time_series/beijing_zoo/archive/beijing_zoo_multivariate_meta_lstm_input_local_legacy.xlsx`

说明：

- 当前正式使用的是整理后的专题底表；
- `archive` 中保留了本地早期版本，便于回溯字段差异。

### 10.4 结果定位

- `实验12` 适合作为正文中的正式动态解释模型；
- `实验17` 更适合作为案例扩展和方法对比材料；
- 若北京动物园多元时间序列模型的残差白噪声检验未通过，应明确说明其原因主要在于指标可得性有限，尚未把所有相关外生因素纳入模型。
