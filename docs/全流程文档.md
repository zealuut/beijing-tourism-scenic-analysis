# 项目全流程文档

## 1. 项目目标

本项目的目标不是只做“景区评论情感分析”，而是基于北京 20 个景区的游客评论，构建一套**可解释的景区治理诊断流程**。

项目主线分成 4 层：

1. `评论数据层`
   获取景区原始评论，并构建规范样本。
2. `文本识别层`
   用 LDA 提取一级主题，用 LLM 识别二级治理问题标签。
3. `情感评估层`
   用 SnowNLP 生成连续情感得分和三分类情感标签。
4. `治理诊断层`
   用 IPA 识别优先治理主题，用聚类和分层识别景区治理画像与紧迫度。

## 5. 阶段 0：原始评论抓取与整理

### 5.1 目标

从携程抓取每个景区的原始评论，并保留以下核心字段：

- 评论文本
- 评分
- 时间

### 5.2 抓取要求

- 每个景区按时间顺序抓到上限
- 只保留评分、时间、评论都齐全的记录
- 先按指定 20 个景区抓取
- 如某景区不满足字段完整性或样本量要求，则在同类别中替换

### 5.3 结果落地

原始分景区文件最终存放在：

- `data\景区数据`

每个文件名为景区中文名，例如：

- `故宫博物院.csv`
- `北京动物园.csv`

## 6. 阶段 1：基础样本表（实验1）

对应脚本：

- `scripts/01_build_experiment1_rawdata.py`

### 6.1 目的

从 20 个景区的原始评论中，构建一个统一基础样本表，保证后续实验的景区口径一致。

### 6.2 方法

- 读取 `data\景区数据` 下 20 个景区文件
- 按评论时间排序
- 每个景区最多选取 300 条
- 汇总成一张大表

### 6.3 主要输出

位于：

- `旅游景点分析/data/rawdata`

核心文件：

- `experiment1_景区300条汇总大表.csv`
- `experiment1_景区抽样汇总.csv`

### 6.4 作用

这一阶段主要解决：

- 各景区样本量统一
- 后续实验有一致的基础样本框架

## 7. 阶段 2：LDA-ready 样本表（实验2）

对应脚本：

- `scripts/02_build_experiment2_lda_ready_rawdata.py`

### 7.1 目的

构建一张可直接用于 LDA 的评论大表。

### 7.2 方法

在实验1基础上进一步做预处理：

- 短句过滤
- 分词
- 低频词过滤
- 景区名等干扰词过滤
- 按评分分层抽样
- 每个景区最多 300 条

### 7.3 为什么要按评分分层

因为如果直接随机抽样，高评分评论会占绝对多数，导致后续 LDA 更容易学到“满意/推荐”类表达，而难以稳定提取治理问题主题。

评分分层抽样的作用是：

- 保留评论结构的多样性
- 提高治理问题主题被识别出来的概率

### 7.4 主要输出

位于：

- `旅游景点分析/data/rawdata`

核心文件：

- `experiment2_lda_评分分层300条汇总大表.csv`
- `experiment2_lda_景区汇总.csv`
- `experiment2_lda_评分分层配额明细.csv`
- `experiment2_lda_全局词频表.csv`

### 7.5 作用

这是后续 LDA 的正式输入表。

## 8. 阶段 3：LDA 一级主题（实验3）

对应脚本：

### 8.1 目的

### 8.2 方法

LDA 降噪规则详见：

- `docs/LDA降噪规则说明.md`

### 8.3 当前结果解释

LDA 候选比较后，当前数据能稳定支持的治理主题主要有：

- 票务预约与入园体验
- 现场服务与导览体验
- 交通与开放空间体验

另有一类满意主题：

- 整体满意与感知评价

### 8.4 主要输出

位于：

核心文件：

- `experiment3_lda_candidate_compare.csv`

## 10. 阶段 5：SnowNLP 情感得分（实验5）

对应脚本：

- `scripts/05_run_experiment5_snownlp_sentiment.py`

说明文档：

- `docs/SnowNLP景区领域情感打分说明.md`

### 10.1 目的

生成评论级连续情感得分与三分类情感标签。

### 10.2 为什么不是直接换词表

SnowNLP 不是单纯的词典打分工具，而是情感分类模型。  
因此这里采用的是：

- 用当前景区评论构造领域正负语料
- 重训 SnowNLP
- 再对当前 6000 条样本打分

### 10.3 当前正式输出

- `sentiment_score`
- `sentiment_class`

当前正式三分类规则为：

- `sentiment_score >= 0.85`：`positive`
- `sentiment_score <= 0.10`：`negative`
- 其他：`neutral`

### 10.4 主要输出

位于：

- `旅游景点分析/data/sentiment`

核心文件：

- `experiment5_snownlp_scored.csv`
- `experiment5_snownlp_threshold_compare.csv`
- `experiment5_snownlp_overall_summary.csv`
- `experiment5_snownlp_scenic_summary.csv`
- `experiment5_snownlp_report.md`

### 10.5 作用

这一层用于回答：

- 这条评论整体情绪偏正还是偏负
- 各景区整体情绪表现如何
- 和问题标签结合后，可识别高分低情绪等偏离现象

## 12. 阶段 7：论文口径 IPA（实验7）

对应脚本：

- `scripts/07_run_experiment7_paper_ipa.py`

输出目录：

- `旅游景点分析/outputs/ipa_paper`

根目录说明：

- `outputs/ipa_paper/README.md`

### 12.1 目的

构建一套只依赖当前样本、在论文中可直接解释的 IPA 指标体系。

### 12.2 当前正式指标

对于景区 `s` 和一级主题 `t`：

- `Importance_(s,t) = n_(s,t) / N_s`
- `IssueRate_(s,t) = m_(s,t) / n_(s,t)`
- `NegativeRate_(s,t) = u_(s,t) / n_(s,t)`
- `Dissatisfaction_(s,t) = (IssueRate_(s,t) + NegativeRate_(s,t)) / 2`
- `Performance_(s,t) = 1 - Dissatisfaction_(s,t)`
- `PriorityIndex_(s,t) = Importance_(s,t) * Dissatisfaction_(s,t)`

### 12.3 解释

- `Importance`：游客对该主题关注度
- `Performance`：该主题体验表现
- `PriorityIndex`：一个主题既被大量提及，又表现较差时，其值会更高

### 12.4 主要输出

位于：

- `旅游景点分析/outputs/ipa_paper`

核心内容：

- 根目录：
  - `experiment7_theme_panel.csv`
- `overall/`
  - `experiment7_ipa_overall_matrix.csv`
  - `experiment7_ipa_overall.png`
  - `experiment7_ipa_overall_report.md`
- `by_scenic/`
  - 每景区一个文件夹，内含：
    - `*_ipa_table.csv`
    - `*_ipa.png`
    - `*_ipa_summary.md`

### 12.5 当前正式性

当前论文正式 IPA 结果应以：

- `outputs/ipa_paper`

为准。

## 13. 阶段 8：论文口径聚类与类型比较（实验8）

对应脚本：

- `scripts/08_run_experiment8_paper_clustering.py`

输出目录：

- `旅游景点分析/outputs/clustering_paper`

根目录说明：

- `outputs/clustering_paper/README.md`

### 13.1 目的

在论文正式版 IPA 指标之上，构建**可解释性强**的聚类与分类比较结果。

### 13.2 为什么不把景区原有类别直接放进聚类

因为景区原有类别更适合作为解释变量，而不是聚类输入。  
如果把它编码进模型，聚类会更像复现原有景区类别，而不是发现治理结构。

### 13.3 当前三层结构

这一阶段正式保留 3 层结果：

1. `景区原有类别比较`
2. `治理画像聚类`
3. `治理紧迫度分层`

### 13.4 主聚类特征

主聚类使用的特征包括：

- 各一级治理主题的 `priority_index`
- 各一级治理主题的 `dissatisfaction_index`
- `total_priority_index`
- `mean_dissatisfaction`

### 13.5 聚类方法

- 标准化：`StandardScaler`
- 主算法：`AgglomerativeClustering`
- linkage：`ward`
- 比较 `k = 2 / 3 / 4 / 5`
- 以轮廓系数 `silhouette_score` 选择最优 `k`

### 13.6 主要输出

位于：

- `旅游景点分析/outputs/clustering_paper`

核心文件：

- `experiment8_cluster_features.csv`
- `experiment8_cluster_k_compare.csv`
- `experiment8_cluster_assignments.csv`
- `experiment8_cluster_profiles.csv`
- `experiment8_cluster_scatter.png`
- `experiment8_cluster_dendrogram.png`
- `experiment8_category_compare.csv`
- `experiment8_urgency_tiers.csv`
- `experiment8_cluster_report.md`

### 13.7 论文中如何使用

建议分 3 层写：

- 用景区原有类别比较回答“哪些类型景区更容易出现什么问题”
- 用治理画像聚类回答“哪些景区在问题结构上相似”
- 用治理紧迫度分层回答“哪些景区更需要优先治理”

## 15. 当前项目的最终主线总结

可以把当前项目压缩成一句话：

**先用原始评论构建统一样本，再用 LDA 提取一级治理主题，用 LLM 标注二级治理问题，用 SnowNLP 评估情绪表现，最后用论文口径 IPA 和可解释聚类完成景区治理诊断。**

更完整地说，就是：

1. 抓取景区原始评论
2. 构建 20 景区统一样本
3. 生成 LDA-ready 评论大表
4. 提取一级治理主题
5. 标注二级治理问题
6. 计算情感得分与三分类
7. 用论文口径 IPA 识别优先治理主题
8. 用可解释聚类和紧迫度分层归纳景区治理类型
