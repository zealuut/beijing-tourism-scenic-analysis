# 情感得分与治理标签回归说明

## 1. 这一步放在全文里的位置

在 LDA、LLM 标签和 SnowNLP 之后，项目已经知道：

- 哪些治理主题最突出；
- 每条评论命中了哪些二级治理问题；
- 每条评论的情感强弱和情感类别如何。

回归步骤进一步回答的是：

**11 个二级治理问题分别会把情绪往什么方向推，且这种关系在连续情感得分和离散情感类别上是否一致。**

## 2. 数据来源

回归底表为：

- `data/regression/sentiment_regression.xlsx`

字段包括：

- `sentiment_score`
- `sentiment_class`
- 11 个二级标签哑变量：
  - `tag_staff_service`
  - `tag_service_process`
  - `tag_guide_explanation`
  - `tag_queue_wait`
  - `tag_reservation_entry`
  - `tag_ticket_price`
  - `tag_traffic_access`
  - `tag_facility_hygiene`
  - `tag_crowding`
  - `tag_commercialization`
  - `tag_platform_transaction`

## 3. 方法口径

对应脚本：

- `scripts/13_run_experiment13_sentiment_issue_regression.py`

当前方法分为两部分：

1. 连续型情感结果  
   以 `sentiment_score` 为因变量，采用多元线性回归。

2. 离散型情感结果  
   以 `sentiment_class` 为因变量，采用逻辑回归口径。当前脚本为了匹配三分类结果，使用多项逻辑回归，并以 `positive` 为基准类别。

说明：

- 11 个标签均以哑变量形式入模；
- 连续情感部分使用稳健标准误，避免异方差影响解释。

## 4. 为什么要做这一步

它的作用不是重新做主题识别，而是把“问题结构”和“情绪结果”接起来：

- 如果某个标签在线性回归中显著拉低情感得分，说明它更可能对应体验恶化；
- 如果某个标签在逻辑回归中显著提高负向情绪发生概率，说明它更容易引发投诉型情绪表达；
- 将这一步与 IPA 结合后，可以区分“被频繁提及”和“真正容易引发情绪恶化”的问题。

## 5. 结果解释建议

建议重点看三类信息：

1. 线性回归系数  
   看哪些标签与连续情感得分下降关系最强。

2. 逻辑回归优势比  
   看哪些标签更容易把评论从 `positive` 推向 `neutral` 或 `negative`。

3. 标签出现频率  
   看“高频问题”和“高情绪风险问题”是否重合。

## 7. 论文建议写法

推荐口径：

> 为进一步检验二级治理问题与游客情绪结果之间的关系，本文在主题识别和情感打分基础上，将 11 个二级治理标签作为解释变量，分别对连续型情感得分和离散型情感类别进行回归分析。连续情感得分部分采用多元线性回归，离散情感类别部分采用逻辑回归口径。该步骤的目的不是替代主题识别，而是从统计关系上说明哪些治理问题不仅被频繁提及，而且更可能伴随明显的情绪恶化。
