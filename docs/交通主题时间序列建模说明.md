# 交通主题时间序列建模说明

## 1. 这一步解决什么问题

前面的 IPA 与聚类已经可以识别：

- `交通与开放空间体验` 是重点治理主题之一；
- 哪些景区在交通相关主题上治理压力更高。

但这些结果仍是横截面诊断，尚未回答：

**交通主题的压力在时间上是否有稳定规律，以及高压日是如何形成和累积的。**

因此，本步骤引入北京交通拥堵指数时间序列，对交通主题进行动态量化。

## 2. 数据来源

专题原始数据位于：

- `data/time_series/traffic/beijing_traffic_congestion_2023_2024.xlsx`
- `data/time_series/traffic/beijing_traffic_congestion_2019_2024.xlsx`

对应时间跨度分别为：

- `2023-04-01` 至 `2024-05-23`
- `2019-04-01` 至 `2024-07-02`

字段均为：

- `time`
- `rate`

其中 `rate` 表示北京交通拥堵指数。

## 3. 方法结构

### 3.1 SARIMA

对应脚本：

- `scripts/14_run_experiment14_traffic_sarima.py`
- `scripts/15_run_experiment15_traffic_sarima_forecast.py`

用途：

- 识别拥堵指数的季节性与短期惯性；
- 提供统计模型口径下的可解释动态规律；
- 对验证期样本进行外推预测比较。

说明：

- `实验15` 需要额外验证文件 `data/time_series/traffic/beijing_traffic_congestion_validation.xlsx`；
- 当前仓库保留脚本接口，但不强行伪造验证样本。

### 3.2 Meta-LSTM-Attention

对应脚本：

- `scripts/16_run_experiment16_traffic_meta_lstm_attention.py`

用途：

- 在更长时间窗口下捕捉拥堵指数的非线性动态结构；
- 用 attention 机制缓解普通 LSTM 对远期信息影响衰减过快的问题；
- 作为交通主题的深度学习补充模型。

## 4. 为什么同时保留 SARIMA 和 Meta-LSTM-Attention

两类模型回答的问题不同：

- `SARIMA`
  更适合给出统计建模口径下的季节性和短期规律，解释清楚，适合正文。

- `Meta-LSTM-Attention`
  更适合补充非线性动态模式和时滞依赖结构，适合方法扩展、附录或案例补充。

因此，这里不是二选一，而是用：

- `SARIMA` 提供稳健的统计主口径；
- `Meta-LSTM-Attention` 提供更强的动态模式补充。

## 5. 与全文主线的关系

交通主题时间序列建模不替代文本识别，而是承接文本识别结果：

1. 文本层告诉我们交通主题重要；
2. IPA/聚类告诉我们交通主题需要治理；
3. 时间序列层进一步告诉我们交通压力的时间规律与高压条件。

这样就把“主题重要”推进到了“主题何时更容易恶化”。

## 6. 结果定位建议

当前项目建议这样使用：

- SARIMA 结果可作为交通主题的正式动态证据；
- Meta-LSTM-Attention 结果可作为更强预测导向的补充比较；
- 若正文需要强调模型稳健性，优先使用通过残差白噪声检验的模型结果；
- 若正文需要强调机制展示，则可以引用 attention 权重或时滞依赖图，但不将其当作显著性检验本身。

## 7. 输出目录

预期输出目录：

- `outputs/traffic_sarima`
- `outputs/traffic_sarima_forecast`
- `outputs/traffic_meta_lstm_attention`

## 8. 论文建议写法

推荐口径：

> 为对重点治理主题中的“交通主题”进行指标量化，本文进一步引入北京交通拥堵指数时间序列数据，并分别采用 SARIMA 和 Meta-LSTM-Attention 两类模型开展分析。SARIMA 模型用于提炼交通压力的季节性和短期惯性规律，Meta-LSTM-Attention 模型则用于补充非线性动态模式与时滞依赖结构。两类模型共同服务于交通主题的动态解释，而非替代前文基于评论文本得到的主题识别结论。
