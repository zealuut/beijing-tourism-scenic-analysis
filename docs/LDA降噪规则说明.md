# LDA降噪规则说明

## 1. 文档目的

本说明用于记录 `experiment3` 中 LDA 建模前使用的降噪规则。

这里的“降噪”只服务于主题模型，不改动原始评论数据，也不回写 `experiment2_lda_评分分层300条汇总大表.csv`。

## 2. 适用范围

- 适用脚本：`旅游景点分析/scripts/03_run_experiment3_lda.py`
- 适用阶段：LDA 建模前的语料清洗
- 输入数据：`旅游景点分析/data/rawdata/experiment2_lda_评分分层300条汇总大表.csv`

## 3. 降噪原则

目标不是删除“负面评论”或“长评论”，而是尽量剔除会干扰主题抽取的非典型文本，尤其是：

- 攻略式、百科式、介绍式长评论
- 模板生成评论
- 高频重复词灌水评论
- 分词结构异常、重复度过高的评论

## 4. 具体规则

### 4.1 文本长度过滤

仅保留评论正文长度在下列范围内的记录：

- 最短 `8` 字
- 最长 `240` 字

作用是过滤过短碎句，以及明显偏攻略或景点介绍的超长文本。

### 4.2 分词长度过滤

在 `tokens_text` 基础上再次处理后，仅保留：

- `token_count >= 5`
- `token_count <= 60`

作用是避免词太少导致主题信息不足，也避免词太多导致评论更像说明文或拼接文本。

### 4.3 词汇多样性过滤

仅保留：

- `unique_token_ratio >= 0.45`

其中：

- `unique_token_ratio = 去重后 token 数 / 总 token 数`

作用是排除重复词太多、表达结构过于单一的评论。

### 4.4 单词重复占比过滤

仅保留：

- `max_single_token_share <= 0.34`

其中：

- `max_single_token_share = 单个 token 最大重复次数 / 总 token 数`

作用是过滤“好好好好好”“推荐推荐推荐”这类由单个词反复堆叠形成的噪声文本。

### 4.5 额外停用词过滤

在实验 2 已完成分词和低频词过滤的基础上，实验 3 又额外去掉一批容易把主题带偏的词，例如：

- `历史`
- `建筑`
- `皇家`
- `园林`
- `古代`
- `位于`
- `游客`
- `看到`
- `感受`
- `今天`
- `樱花`
- `红叶`
- `动物`
- `孩子`

这一层的目标是减少“景点介绍文”和“季节打卡文”对主题模型的干扰，让主题更聚焦游客体验与治理问题。

### 4.6 模板评论识别

若评论中出现明显模板话术，则直接判定为噪声，例如：

- `以下是对`
- `我给你写了`
- `直接复制`
- `简短版`
- `详细版`
- `点评通用`
- `游玩点评`
- `相关照片`

这类文本通常不是自然评论，而更像生成式内容、整理稿或平台外部拷贝文本。

### 4.7 重复模式识别

若评论匹配下列模式，也会被剔除：

- 某个长度为 `2-8` 的短语连续重复至少 4 次
- `好 / 棒 / 赞 / 差 / 啊` 等单字被连续重复至少 7 次

作用是去掉明显灌水、复制粘贴或情绪词堆叠文本。

## 5. 执行结果解释

在当前这轮实验中，LDA 最终输入语料从原始 `6000` 条降到 `5380` 条。

这说明被过滤掉的主要是：

- 模板化评论
- 攻略式评论
- 重复度过高评论
- 对主题建模贡献较弱的异常文本

## 6. 方法边界

需要注意：

- 该降噪规则只用于 LDA，不影响原始评论保存
- 该规则是“主题建模友好型过滤”，不是通用清洗标准
- 若后续目标改为情感分析、分类标注或案例展示，过滤阈值不一定应保持一致

## 7. 后续建议

如果后续还要继续优化 LDA，可优先从这几个方向微调：

- 继续扩充模板评论识别词表
- 对“景点百科式介绍文”单独加规则
- 对极高相似度评论做去重
- 按景区类型分别建立更细的停用词表
